Структура доменного имени

Дата публикации: 12 января 2026 года

Доменное имя — это человекочитаемый адрес сайта в интернете, например: github.com или 31014000qwerty-jpg.github.io. Оно состоит из нескольких уровней (или частей), иерархически организованных справа налево.

Уровни доменного имени

Рассмотрим пример домена: 31014000qwerty-jpg.github.io

  • Корневой домен (root domain) — обозначается точкой (.) в конце полного доменного имени. Обычно не отображается, но технически существует.
  • Домен верхнего уровня (TLD — Top-Level Domain): io — указывает на категорию или географическое происхождение (в данном случае — территория Британских Индийских океанических территорий, но часто используется как общедоступный TLD).
  • Домен второго уровня (SLD — Second-Level Domain): github — основное имя, зарегистрированное организацией или пользователем.
  • Поддомен (Subdomain): 31014000qwerty-jpg — дополнительный уровень, созданный владельцем домена второго уровня для организации контента или сервисов.

Таким образом, полное доменное имя читается справа налево по уровням иерархии:
. → io → github → 31014000qwerty-jpg

В случае GitHub Pages, когда вы создаёте репозиторий с именем <username>.github.io, GitHub автоматически предоставляет вам поддомен вида <username>.github.io, где:

  • io — TLD,
  • github — SLD,
  • 31014000qwerty-jpg — ваш поддомен (имя пользователя).
